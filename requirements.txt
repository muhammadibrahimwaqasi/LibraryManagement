================================================================================
LIBRARY MANAGEMENT SYSTEM - REQUIREMENTS
================================================================================

1. FUNCTIONAL REQUIREMENTS
================================================================================

USER MANAGEMENT:
- Add/Edit/Delete users
- Name, email, role fields
- Roles: user, student, admin, librarian, teacher

BOOK MANAGEMENT:
- Add/Edit/Delete books
- Title, author, category, quantity fields
- Search by title or category
- Create new author/category inline when adding book
- Track available quantity

AUTHORS:
- Add/Edit/Delete authors
- View all authors

CATEGORIES:
- Add/Edit/Delete categories
- View all categories

ISSUE/RETURN:
- Issue books to registered users
- Return books
- Track issue history
- Calculate late fines (PKR 500/day)

RESERVATIONS:
- Create/Delete reservations
- View all reservations

FINES:
- View/Delete fines

LIBRARIANS:
- Add/Delete librarians

NOTIFICATIONS:
- View/Delete notifications

================================================================================
2. TECHNICAL REQUIREMENTS
================================================================================

BACKEND:
- Node.js + Express.js
- PostgreSQL (Neon)
- MVC architecture
- RESTful API

FRONTEND:
- React.js
- React Router DOM
- Functional components with hooks
- Responsive design

================================================================================
4. BUSINESS RULES
================================================================================

- Book can only be issued if available > 0
- Due date = 14 days from issue date
- Late fine = PKR 500 per day
- Email must be unique
- Author and category optional for books

================================================================================
5. USER ROLES
================================================================================

- user     (default)
- student
- admin
- librarian
- teacher
